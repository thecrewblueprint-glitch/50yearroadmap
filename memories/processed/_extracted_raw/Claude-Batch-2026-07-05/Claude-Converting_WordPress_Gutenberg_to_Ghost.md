     8	# Reading Uploaded Files
     9	
    10	## Why this skill exists
    11	
    12	When a user uploads a file in claude.ai, Claude Desktop, or Cowork,
    13	the file is written to `/mnt/user-data/uploads/<filename>` and you are told the path
    14	in an `<uploaded_files>` block. **The content is not in your context.**
    15	You must go read it.
    16	
    17	The naive thing — `cat /mnt/user-data/uploads/whatever` — is wrong for
    18	most files:
    19	
    20	- On a PDF it prints binary garbage.
    21	- On a 100MB CSV it floods your context with rows you will never use.
    22	- On a DOCX it prints the raw ZIP bytes.
    23	- On an image it does nothing useful at all.
    24	
    25	This skill tells you the right first move for each type, and when to
    26	hand off to a deeper skill.
    27	
    28	## General protocol
    29	
    30	1. **Look at the extension.** That is your dispatch key.
    31	2. **Stat before you read.** Large files need sampling, not slurping.
    32	   ```bash
    33	   stat -c '%s bytes, %y' /mnt/user-data/uploads/report.pdf
    34	   file /mnt/user-data/uploads/report.pdf
    35	   ```
    36	3. **Read just enough to answer the user's question.** If they asked
    37	   "how many rows are in this CSV", don't load the whole thing into
    38	   pandas — `wc -l` gives a fast approximation (it counts newlines,
    39	   not CSV records, so it may over-count if quoted fields contain
    40	   embedded newlines).
    41	4. **If a dedicated skill exists, go read it.** The table below tells
    42	   you when. The dedicated skills cover editing, creating, and advanced
    43	   operations that this skill does not.
    44	
    45	## `extract-text`
    46	
    47	For docx, odt, epub, xlsx, pptx, rtf, and ipynb the first move is
    48	`extract-text <file>`. It emits markdown for docx/odt/epub (headings,
    49	bold, lists, links, tables), tab-separated rows under `## Sheet:`
    50	headers for xlsx, text under `## Slide N` headers for pptx, fenced
    51	code cells for ipynb, and plain text for rtf. Pass `--format <fmt>`
    52	when the extension is wrong or absent (e.g., `--format xlsx` on an
    53	`.xlsm`). If it errors on a file, `pandoc <file> -t plain` is a
    54	fallback; for xlsx/pptx, fall back to the dedicated skill's
    55	Python-based approach (openpyxl / python-pptx).

    83	## PDF
    84	
    85	**Never** `cat` a PDF — it prints binary garbage.
    86	
    87	Quick first move — get the page count and check if text is extractable:
    88	
    89	```bash
    90	pdfinfo /mnt/user-data/uploads/report.pdf
    91	pdftotext -f 1 -l 1 /mnt/user-data/uploads/report.pdf - | head -20
    92	```
    93	
    94	Then peek at the text content:
    95	
    96	```python
    97	from pypdf import PdfReader
    98	r = PdfReader("/mnt/user-data/uploads/report.pdf")
    99	print(f"{len(r.pages)} pages")
   100	print(r.pages[0].extract_text()[:2000])
   101	```
   102	
   103	For anything beyond a quick peek — figures, tables, attachments,
   104	forms, scanned PDFs, visual inspection, or choosing a reading strategy
   105	— go read `/mnt/skills/public/pdf-reading/SKILL.md`. It covers
   106	content inventory, text extraction vs. page rasterization, embedded
   107	content extraction, and document-type-aware reading strategies.
   108	
   109	For PDF form filling, creation, merging, splitting, or watermarking,
   110	go read `/mnt/skills/public/pdf/SKILL.md`.

   128	## XLSX / XLS / spreadsheets
   129	
   130	The `xlsx` skill covers formulas, formatting, charts, creating. Read
   131	it if you need any of those. For a quick look at an `.xlsx`:
   132	
   133	```bash
   134	extract-text /mnt/user-data/uploads/data.xlsx | head -100
   135	```
   136	
   137	For `.xlsm`, add `--format xlsx` (same zip structure; only the
   138	extension differs). When you need a structured preview in Python:
   139	
   140	```python
   141	from openpyxl import load_workbook
   142	wb = load_workbook("/mnt/user-data/uploads/data.xlsx", read_only=True)
   143	print("Sheets:", wb.sheetnames)
   144	ws = wb.active
   145	for row in ws.iter_rows(max_row=5, values_only=True):
   146	    print(row)
   147	```
   148	
   149	`read_only=True` matters — without it, openpyxl loads the entire
   150	workbook into memory, which breaks on large files. Do not trust
   151	`ws.max_row` in read-only mode: many non-Excel writers omit the
   152	dimension record, so it comes back `None` or wrong. If you need a row
   153	count, iterate or use pandas.
   154	
   155	**Legacy `.xls`** — openpyxl raises `InvalidFileException`. Use:
   156	
   157	```python
   158	import pandas as pd
   159	df = pd.read_excel("/mnt/user-data/uploads/old.xls", engine="xlrd", nrows=5)
   160	```
   161	
   162	**`.ods` (OpenDocument)** — openpyxl also rejects this. Use:
   163	
   164	```python
   165	import pandas as pd
   166	df = pd.read_excel("/mnt/user-data/uploads/data.ods", engine="odf", nrows=5)
   167	```

   187	## CSV / TSV
   188	
   189	**Do not** `cat` or `head` these blindly. A CSV with a 50KB quoted cell
   190	in row 1 will wreck your `head -5`. Use pandas with `nrows`:
   191	
   192	```python
   193	import pandas as pd
   194	df = pd.read_csv("/mnt/user-data/uploads/data.csv", nrows=5)
   195	print(df)
   196	print()
   197	print(df.dtypes)
   198	```
   199	
   200	Approximate row count without loading (over-counts if the file has
   201	RFC-4180 quoted newlines — the same quoted-cell case this section
   202	warned about above):
   203	
   204	```bash
   205	wc -l /mnt/user-data/uploads/data.csv
   206	```
   207	
   208	Full analysis only after you know the shape:
   209	
   210	```python
   211	df = pd.read_csv("/mnt/user-data/uploads/data.csv")
   212	print(df.describe())
   213	```
   214	
   215	TSV: same, with `sep="\t"`.

   219	## JSON / JSONL
   220	
   221	Structure first, content second:
   222	
   223	```bash
   224	jq 'type' /mnt/user-data/uploads/data.json
   225	jq 'if type == "array" then length elif type == "object" then keys else . end' /mnt/user-data/uploads/data.json
   226	```
   227	
   228	(`keys` errors on scalar JSON roots — a bare `"hello"` or `42` is valid
   229	JSON per RFC 7159 — so guard the branch.)
   230	
   231	Then drill into what the user actually asked about.
   232	
   233	JSONL (one object per line) — do **not** `jq` the whole file; work line
   234	by line:
   235	
   236	```bash
   237	head -3 /mnt/user-data/uploads/data.jsonl | jq .
   238	wc -l /mnt/user-data/uploads/data.jsonl
   239	```

   243	## Images (JPG / PNG / GIF / WEBP)
   244	
   245	**You can already see uploaded images.** They are injected into your
   246	context as vision inputs alongside the `<uploaded_files>` pointer. You
   247	do not need to read them from disk to describe them.
   248	
   249	The disk copy is only needed if you are going to **process** the image
   250	programmatically:
   251	
   252	```python
   253	from PIL import Image
   254	img = Image.open("/mnt/user-data/uploads/photo.jpg")
   255	print(img.size, img.mode, img.format)
   256	```
   257	
   258	For OCR on an image (text extraction, not description):
   259	
   260	```python
   261	import pytesseract
   262	print(pytesseract.image_to_string(img))
   263	```
   264	
   265	Note: the client resizes images larger than 2000×2000 down to that
   266	bound and re-encodes as JPEG before upload, so the disk copy may not
   267	be the user's original bytes. For most processing this doesn't matter;
   268	if the user is asking about original-resolution pixel data, flag it.

   272	## Archives (ZIP / TAR / TAR.GZ)
   273	
   274	**List first. Extract never — unless the user explicitly asks.**
   275	Archives can be huge, contain path traversal, or nest forever.
   276	
   277	```bash
   278	unzip -l /mnt/user-data/uploads/bundle.zip
   279	tar -tf /mnt/user-data/uploads/bundle.tar
   280	```
   281	
   282	GNU tar auto-detects compression — `tar -tf` works on `.tar`,
   283	`.tar.gz`, `.tar.bz2`, `.tar.xz` alike. Don't hard-code `-z`.
   284	
   285	If the user wants one file from inside, extract just that one:
   286	
   287	```bash
   288	unzip -p /mnt/user-data/uploads/bundle.zip path/inside/file.txt
   289	```
   290	
   291	**Standalone `.gz`** (not a tar) compresses a single file — there is
   292	no manifest to list. Just peek at the decompressed content:
   293	
   294	```bash
   295	zcat /mnt/user-data/uploads/data.json.gz | head -50
   296	```

Written for the Safety Vault as a living document — print-ready, web-publishable, and designed to be distributed at call time.</p>\n  <!-- /wp:paragraph -->\n\n  <!-- wp:group {\"className\":\"tcb-manifesto-body\",\"layout\":{\"type\":\"constrained\",\"contentSize\":\"780px\"}} -->\n  <div class=\"wp-block-group tcb-manifesto-body\">\n\n    <!-- wp:quote {\"className\":\"tcb-manifesto-opening-quote\"} -->\n    <blockquote class=\"wp-block-quote tcb-manifesto-opening-quote\">\n      <p>This industry does not forgive carelessness. It rewards preparation, punishes distraction, and remembers the names of those who were not ready. This Manifesto is not a liability form. It is a code of conduct written by crew, for crew. Read it. Own it. Work by it every single day you step onto a stage.</p>\n    </blockquote>\n    <!-- /wp:quote -->\n\n    <!-- wp:heading {\"level\":3,\"className\":\"tcb-manifesto-h2\"} -->\n    <h3 class=\"wp-block-heading tcb-manifesto-h2\">Who Is a Ground Hand? <span class=\"tcb-tag\">FOUNDATION</span></h3>\n    <!-- /wp:heading -->\n\n    <!-- wp:paragraph -->\n    <p>A Ground Hand is the backbone of any production load-in or load-out. You are the first on deck and sometimes the last to leave. You move freight, build decks, spot forklifts, chase cables, wrap truss, and keep the show alive from the ground up. The role is entry-level by title, never by standard. Every Head Carpenter, Lighting Director, and Production Manager on this planet once held the position you are in right now. How they performed it determined everything that followed.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:paragraph -->\n    <p>Your responsibility on a ground crew is threefold: protect yourself, protect your crew, and protect the gear and venue — in that exact order. Nothing ships, nothing flies, and no show opens if someone goes home in an ambulance.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:heading {\"level\":3,\"className\":\"tcb-manifesto-h2\"} -->\n    <h3 class=\"wp-block-heading tcb-manifesto-h2\">THE DEATH ZONE <span class=\"tcb-tag tcb-tag--red\">CRITICAL</span></h3>\n    <!-- /wp:heading -->\n\n    <!-- wp:group {\"className\":\"tcb-death-zone-box\"} -->\n    <div class=\"wp-block-group tcb-death-zone-box\">\n      <!-- wp:paragraph {\"className\":\"tcb-death-zone-box__text\"} -->\n      <p class=\"tcb-death-zone-box__text\"><strong class=\"tcb-red\">THE DEATH ZONE IS DEFINED AS ANY AREA DIRECTLY BELOW AN ACTIVE LIFT, FLY, OR RIGGING OPERATION.</strong> This includes the perimeter zone — typically a minimum radius of 1.5× the height of the lift point. If a motor is running, a chain is moving, or a truss is being flown, you do not enter that zone without a direct, verbal authorization from the Head Rigger or department lead. No exceptions. No \"just grabbing something.\" No shortcuts.</p>\n      <!-- /wp:paragraph -->\n    </div>\n    <!-- /wp:group -->\n\n    <!-- wp:paragraph -->\n    <p>Production rigging operates with loads exceeding thousands of pounds at heights that make falls or drops instantly fatal. A single loose connector, a worn shackle pin, or an improperly seated motor hook can send hundreds of pounds of truss, fixtures, and hardware into freefall with zero warning. The physics are not survivable. The only protection is distance and discipline.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:paragraph -->\n    <p>When working near any overhead operation: identify your escape route before the lift begins. Never stand with your back to an active rigging zone. Know where the dead zones are — columns, walls, and far-side positions that take you out of the drop radius. Make it habit. Make it reflex.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:heading {\"level\":3,\"className\":\"tcb-manifesto-h2\"} -->\n    <h3 class=\"wp-block-heading tcb-manifesto-h2\">PPE — Personal Protective Equipment <span class=\"tcb-tag\">MANDATORY</span></h3>\n    <!-- /wp:heading -->\n\n    <!-- wp:paragraph -->\n    <p>PPE is not optional. It is not a suggestion from a safety officer who doesn't understand production pace. It is the minimum barrier between you and an injury that ends careers. Know your gear. Wear it correctly.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:columns {\"className\":\"tcb-ppe-grid\"} -->\n    <div class=\"wp-block-columns tcb-ppe-grid\">\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Hard Hat / Helmet</strong><br>Class E rated. Inspect before every call. Replace after any impact, even if no visible damage. Your brain is not replaceable.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Steel-Toed Boots</strong><br>ASTM F2413 rated minimum. No sneakers, no exceptions. Cases, motors, and truss sections will find your feet.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Hi-Visibility Vest</strong><br>Required anytime forklifts or heavy equipment are operating in the same venue space. Be seen.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n    </div>\n    <!-- /wp:columns -->\n\n    <!-- wp:columns {\"className\":\"tcb-ppe-grid\"} -->\n    <div class=\"wp-block-columns tcb-ppe-grid\">\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Gloves</strong><br>Cut-resistant, grip-positive. Essential for cable runs, truss sections, and any rigging hardware handling.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Hearing Protection</strong><br>For any pre-show loud test, generator rooms, or sustained loud power tool use. Hearing loss is permanent.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n      <!-- wp:column {\"className\":\"tcb-ppe-item\"} -->\n      <div class=\"wp-block-column tcb-ppe-item\">\n        <!-- wp:paragraph --><p><strong class=\"tcb-ppe-label\">Fall Arrest Harness</strong><br>Required for any work above 6 feet off deck. Inspect all points — stitching, buckles, D-rings — before each climb.</p><!-- /wp:paragraph -->\n      </div><!-- /wp:column -->\n    </div>\n    <!-- /wp:columns -->\n\n    <!-- wp:heading {\"level\":3,\"className\":\"tcb-manifesto-h2\"} -->\n    <h3 class=\"wp-block-heading tcb-manifesto-h2\">Communication Protocols <span class=\"tcb-tag\">OPERATIONS</span></h3>\n    <!-- /wp:heading -->\n\n    <!-- wp:paragraph -->\n    <p>More accidents in live production happen from miscommunication than from equipment failure. A motor operator who doesn't hear \"clear\" and a ground crew member who assumes the call was given can produce identical, tragic outcomes. Communication on a ground crew is formal. It is structured. It is never assumed.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:list {\"ordered\":false,\"className\":\"tcb-protocol-list\"} -->\n    <ul class=\"tcb-protocol-list\">\n      <li><strong>Call and Confirm:</strong> No rigging or overhead movement begins without an audible two-way confirmation. The call goes out (\"Flying truss 1 — clear the deck\"), and only a verbal all-clear from every department on deck authorizes movement to begin.</li>\n      <li><strong>Radio Discipline:</strong> Use your comms clearly and concisely. State your name, department, and message. Avoid cross-talk during active lifts. \"Position\" calls during motor runs are not casual conversation — they are safety coordinates.</li>\n      <li><strong>Hand Signals:</strong> Know ASME B30.3 hand signal standards — thumb up (hoist), thumb down (lower), closed fist (emergency stop). When radio fails, hand signals keep the crew alive. Practice them until they are reflexive.</li>\n      <li><strong>Stop Work Authority:</strong> Every crew member — from the newest local hire to the production manager — holds full stop work authority. They are the definition of craft.</p>\n    <!-- /wp:paragraph -->\n\n    <!-- wp:paragraph {\"className\":\"tcb-closing-line\"} -->\n    <p class=\"tcb-closing-line\">Protect the crew.
