# 00 — Overview

**Status:** Documented.

## What it is

Personal Operations — the **Operations Command Center** — is the repo-backed,
AI-assisted system that runs the whole initiative. It organizes businesses, apps,
goals, daily reports, completed work, future plans, long-term vision, and deep
roadmap context into one auditable place. It is the **decision framework and
execution system** for everything else in this repo.

## The core question

The system is designed to answer one practical question:

> What is the next thing I should probably focus on without losing the rest of
> the vision?

Everything — the data model, the dashboard, the pipeline, the governance — exists
to keep answering that question honestly as reality changes.

## Core principles

1. **Chaos goes in. Auditable roadmap structure comes out.**
2. **Flexible detours without destroying linear progression.**
3. **Raw memory remains evidence, not public output.**
4. **Deep context influences direction, but is not published raw.**
5. **Moved-later items are preserved, not deleted.**

## What it is made of

- **A data model** — `roadmap.json`, the single source of truth (doc 01).
- **A dashboard** — live views over that data (doc 02).
- **An AI pipeline** — turns raw evidence into reviewable proposals (doc 03).
- **Governance** — the rules, log, and safety controls that keep multiple AI
  collaborators consistent and the repo public-safe (doc 04).

## Current stage

Mature and operational (roadmap `personal-operations` ~65%). The dashboard is
live, governance and the pipeline are established, and canonical documentation
for all companies is being built. Remaining work is polish (doc 05).

## Where it sits in the ecosystem

This branch is the **meta-layer** — the operating system the other four
companies (Deadhang Labor, The Crew Blueprint, Production Atlas, Contractor
Tools) are planned and tracked within. It doesn't sell anything; it keeps the
whole enterprise coherent and moving.
