# 08 — Growth Model

**Status:** Documented.

## The strategic shift

The Crew Blueprint has crossed an important line: it is **no longer blocked by
software development.** The foundational architecture and workflow decisions are
largely made (vision, course architecture, content workflow, brand, legal
philosophy, website shell, plugin architecture, AI workflow). The bottleneck is
now **producing and organizing high-quality training content** and building the
**governance and documentation** to support the platform long term.

## The path

1. **Content-first (now).** Produce canonical course packets (doc 07), continue
   **Stagehand Fundamentals**, and build additional foundational courses. Keep the
   website a lightweight presentation layer (doc 05).
2. **Foundational library.** Organize the resource library by discipline/pathway
   (doc 01, roadmap `cb-2`) and grow enough foundational content to make the
   resource genuinely useful (roadmap `cb-4`).
3. **Next-generation platform (later).** Once content volume justifies it, build
   the deferred platform (Laravel/AWS/DB/RAG/search/accounts/mobile — doc 05).
   Content demand pulls the platform into existence, not the reverse.

## Highest-priority active work

- Produce canonical course packets.
- Build the permanent documentation repository (this knowledge base).
- Complete governance.
- Continue **Stagehand Fundamentals**.
- Build additional courses.
- Keep the website lightweight until content justifies the next platform.

## Long-term expansion

Over time, the library grows to cover every major production discipline and the
professional skills around it (full list in doc 01) — lighting, audio, video,
rigging, carpentry, stage management, touring, corporate events, leadership, OSHA
awareness, contractor education, and mental-health awareness for production
workers.
